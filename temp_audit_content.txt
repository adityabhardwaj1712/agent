2. Complete Error & Bug List (Every Line Referenced)
Every bug found, ordered by severity, with exact file and line numbers.
2.1 Critical Bugs — App Will Crash or Data Will Leak
BUG-001 · llm.py · Lines 54-60 · _groq_client AttributeError CRASH
The _groq method references self._groq_client but this attribute is NEVER declared in __init__(). The __init__ only declares _openai_client, _anthropic_client, _gemini_client. Any call to a Groq model (llama3-*, mixtral-*) will crash with AttributeError: 'LLMService' object has no attribute '_groq_client'.
def __init__(self):
    self._openai_client = None      # ✓ declared
    self._anthropic_client = None   # ✓ declared
    self._gemini_client = None      # ✓ declared
    # _groq_client NEVER declared → crash on first Groq call
Fix: Add self._groq_client: Any = None to __init__.
BUG-002 · agent_worker.py · Lines 156-163 · task.output field does not exist
The worker writes task.output = output and task.output = f'ERROR: {error}' but the Task model (task.py) has no column named 'output'. The model has output_data and result columns. This causes a silent data loss — the result is never saved to DB, and SQLAlchemy may raise an AttributeError.
# agent_worker.py:161  →  task.output = output     ← WRONG
# agent_worker.py:177  →  task.output = 'ERROR:...' ← WRONG
# task.py model has:   result, output_data — NOT 'output'
Fix: Change task.output to task.result in _mark_completed() and _mark_failed().
BUG-003 · agent_worker.py · Line 87 · agent_delegate imported but never defined
The worker calls await agent_delegate(...) at line 87 inside _delegate_tool(), but agent_delegate is never imported anywhere in agent_worker.py. The import line 'from app.services.event_bus import event_bus' does not include agent_delegate. This crashes the entire delegate_to_agent tool with NameError.
from app.services.event_bus import event_bus
# agent_delegate is NOT imported — NameError on first delegation
Fix: Add agent_delegate to the event_bus import, or import it from where it is defined.
BUG-004 · agent_worker.py · Lines 297-301 · user_id and goal_id undefined in retry block
When a task fails and retry logic runs (lines 297-301), the code references user_id and goal_id which are only defined in an inner scope within process_one_task(). The retry block in run_worker() has no access to these variables. This causes NameError: name 'user_id' is not defined when any task is retried.
# run_worker() retry block:
await orchestrator.enqueue_task(... user_id=user_id ...)  # user_id not in scope!
await orchestrator.enqueue_task(... goal_id=goal_id ...)  # goal_id not in scope!
Fix: Extract user_id and goal_id from task_dict before passing to enqueue_task.
BUG-005 · tasks.py · All routes · No authentication — any user accesses all tasks
Every route in app/api/v1/tasks.py (GET /, POST /, GET /{task_id}, POST /goals, GET /goals, GET /goals/{goal_id}) uses get_db but NOT get_current_user. Any anonymous request can list all tasks, create tasks, and view goals. The user_id is hardcoded to 'demo-user' on every route.
# tasks.py lines 15,22,30,37,61,69 — ALL routes missing auth:
async def get_tasks(db: AsyncSession = Depends(get_db)):        # ← no auth
    return await task_service.send_task(db, task, user_id="demo-user")  # ← hardcoded
Fix: Add current_user: User = Depends(get_current_user) to all task routes and use current_user.user_id.
BUG-006 · workflows.py · All routes · No authentication + hardcoded user_id
All 4 routes in app/api/v1/workflows.py (GET /, POST /, POST /run, GET /{name}) have no authentication. The save and run routes hardcode user_id='demo-user'. Any unauthenticated user can read and overwrite any workflow.
# workflows.py lines 43, 56:
user_id="demo-user"   # ← hardcoded, no auth
Fix: Same as BUG-005 — add get_current_user dependency.
BUG-007 · goals.py · All routes · No authentication + hardcoded 'test-user'
All routes in app/api/v1/goals.py use user_id = 'test-user' (not even 'demo-user' — a different inconsistent stub). No get_current_user dependency anywhere. Background tasks are spawned without any user context validation.
# goals.py line 33:
user_id = "test-user"   # ← hardcoded string literal
Fix: Add authentication and use current_user.user_id.
BUG-008 · approvals.py · All routes · No authentication — anyone can approve/reject
GET /approvals, POST /approvals/{id}/approve, POST /approvals/{id}/reject all have no authentication. Any anonymous user on the internet can approve or reject Human-in-the-Loop approval requests, which bypasses the entire safety mechanism of the platform.
# approvals.py lines 25,30,42 — no get_current_user anywhere
Fix: Add authentication to all 3 routes.

BUG-009 · auto_mode.py, deployment.py, developer.py, query.py · Empty files

BUG-010 · models/ · 18 files · datetime.utcnow() deprecated in Python 3.12+
The following model files use datetime.datetime.utcnow() as column defaults, which is deprecated since Python 3.12 and removed in Python 3.14. The app targets Python 3.12 (confirmed by .pyc file naming). This generates DeprecationWarning in every database write:

default=datetime.datetime.utcnow
# CORRECT:

BUG-011 · task_service.py · Lines 23,47,62 · MD5 used for security-sensitive hashing
Task deduplication and cache keys use MD5 (hashlib.md5). MD5 is cryptographically broken with known collision attacks. A malicious user could craft two different payloads with the same MD5 hash, bypassing the deduplication check and getting a cached result from a different task.
# task_service.py lines 23, 47, 62
return hashlib.md5(content.encode()).hexdigest()  # ← collision-vulnerable
Fix: Replace all hashlib.md5 with hashlib.sha256.
BUG-012 · agent_service.py · seed_builtin_agents · Duplicate docstring
The function seed_builtin_agents has the docstring 'Ensure the built-in agents exist for this owner.' written TWICE (lines ~43 and ~60). The second docstring is a floating string literal inside the function body after the user-creation block. While not a crash, it indicates copy-paste error and confuses documentation tools.
BUG-013 · agent_service.py · register_agent · Returns dict not ORM model
The register_agent function returns a hand-crafted dict but the API route uses response_model=AgentResponse (a Pydantic schema). FastAPI will try to validate the dict against AgentResponse. If any field name in the dict doesn't match the schema exactly, it silently drops it or raises a validation error. The 'token' field in the returned dict is not in AgentResponse.
BUG-014 · billing_service.py · check_limits · Always returns True
The check_limits method fetches the subscription plan and limits but then unconditionally returns True without actually checking usage against limits. Plan limits are never enforced — any user can run unlimited tasks regardless of their subscription tier.
# billing_service.py check_limits():
limits = self.PLANS[plan_key]["limits"]
return True   # ← limit check completely missing, always passes
BUG-015 · event_bus.py · timestamp using deprecated asyncio.get_event_loop()
event_bus.py line 26 uses asyncio.get_event_loop().time() to generate event timestamps. asyncio.get_event_loop() is deprecated in Python 3.10+ and raises DeprecationWarning in 3.12. Replace with asyncio.get_running_loop().time().
BUG-016 · tasks.py · /goals POST · asyncio.create_task() outside async context
tasks.py line 58 calls asyncio.create_task(autonomous_orchestrator.run_goal(new_goal.goal_id)) directly. If the event loop is not running at the point of the call (e.g. during test execution), this raises RuntimeError. Use BackgroundTasks (FastAPI) or ensure the call is within an async context — goals.py correctly uses BackgroundTasks, but tasks.py does not.
BUG-017 · traces.py · get_flame_graph · Incorrect indentation in for loop
In get_flame_graph (traces.py), the for loop body has incorrect indentation at the 'dur = 0.1' assignment (line with extra space before 'dur'). While Python resolves it, it's inconsistent and will confuse linters. More critically, the function returns CSS variable strings ('var(--g)') as JSON data, which will break any frontend that tries to parse them as real color values.

3. How to Improve Code Proficiency
3.1 Type Safety — Biggest Code Quality Gap
The codebase uses dict, Any, and untyped returns extensively. Adding strict types will catch bugs at development time instead of production runtime.
Run mypy app/ --strict --ignore-missing-imports in CI — it will find ~40+ type errors
Replace all '-> dict' return types with proper TypedDict or Pydantic models
The orchestrator's task_data dict (orchestrator.py line ~65) has 10 keys — define TaskPayload(TypedDict)
Replace all 'context: dict | None' in ToolExecutor with a ToolContext(TypedDict) type
Add py.typed marker file to indicate the package supports type checking
3.2 Testing — Zero Coverage, Highest Risk
There are zero test files in the entire project. This is the single most dangerous gap for a production system.
Add pytest + pytest-asyncio + httpx[asyncio] to requirements-dev.txt
Use testcontainers-python to spin up real PostgreSQL + Redis for integration tests
Write unit tests for: task_service.py (dedup logic), circuit_breaker.py (state transitions), billing_service.py (limit checks)
Write API endpoint tests using FastAPI's TestClient for all auth-protected routes
Set minimum 70% coverage with coverage.py --fail-under=70 in CI
Add Playwright E2E tests for: register → login → create agent → run task → view result
3.3 Dependency Management
requirements.txt has zero version pins. This is a reproducibility and supply-chain security risk.
Pin every direct dependency: fastapi==0.115.x, sqlalchemy==2.0.x, redis==5.x
Generate requirements.lock using pip-compile or uv pip compile
Create requirements-dev.txt for pytest, mypy, ruff, black, pre-commit
Run pip-audit in CI to check for known CVEs in dependencies
Consider migrating to pyproject.toml with uv or poetry for modern packaging
3.4 Linting & Formatting
Add ruff (fast linter): ruff check app/ --select E,F,W,B,I,N,UP — UP catches the utcnow() deprecated calls automatically
Add black for formatting: black app/ --line-length 100
Add pre-commit hooks so linting runs on every commit (zero CI cost)
Fix all existing ruff violations — estimated ~80-100 violations based on patterns found
For TypeScript: add ESLint with @typescript-eslint/strict and prettier
3.5 Database Optimization
Add missing indexes: Task.status, Task.created_at, AuditLog.user_id — analytics queries full-scan now
Add CHECK constraint on Task.status: ('queued','processing','completed','failed','pending_approval')
The Alembic migration 0001_initial_reset.py is 800+ lines — split future changes into small named migrations
Add pgvector CREATE EXTENSION to the migration script so setup is fully automated
Use SQLAlchemy selectinload() for Agent→Task relationships to avoid N+1 queries in analytics
3.6 Security Hardening
Implement refresh tokens — 1-week access tokens with no revocation is dangerous
Add rate limiting to /auth/login (max 5/min per IP) to prevent brute force
Add Stripe webhook signature verification in webhooks.py (currently commented out)
The guardian validate endpoint (agents.py) uses simple string matching — use a proper SQL injection detection library
Add Content-Security-Policy and X-Frame-Options headers in MetricsMiddleware
