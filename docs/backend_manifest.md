## 🏗️ Core Architecture

- **Engine**: [FastAPI](https://fastapi.tiangolo.com/) (High-performance Async Python)
- **Database**: [PostgreSQL](https://www.postgresql.org/) with [pgvector](https://github.com/pgvector/pgvector) for AI memory.
- **ORM**: [SQLAlchemy 2.0](https://www.sqlalchemy.org/) (Async-first)
- **Task Queue**: [ARQ](https://github.com/samuelcolvin/arq) (Redis-backed async worker)
- **Cache**: [Redis](https://redis.io/) (Result caching, Rate limiting, Event Bus)
- **Logging**: [Loguru](https://github.com/Delgan/loguru) (Structured tactical logging)

## ⚡ Mission-Critical Features

### 1. 🩸 Self-Healing Database (`SyncManager`)
The backend features an autonomous migration system located in `app/db/migrations.py`.
- **Harden Schema**: Automatically detects and injects missing tactical columns (e.g., `org_id`, `reputation_score`).
- **Provisioning**: Seeds the primary 42-agent roster on startup to ensure immediate operational readiness.

### 2. 🧠 Autonomous Mission Orchestration
The Orchestrator (`app/services/orchestrator.py`) and DAG Engine (`app/services/dag_engine.py`) allow:
- **Priority Routing**: Semantic detection of mission urgency (URGENT, CRITICAL, NORMAL).
- **Smart Agent Selection**: Auto-selection of agents based on a performance-to-cost utility function.
- **Decomposition**: Complex goals are broken into multi-stage DAG nodes.

### 3. 🛡️ Industrial Security & Reliability
- **Guardian System**: Validates all incoming payloads for malicious content or tactical breaches.
- **Circuit Breakers**: Prevents cascading failures in external AI provider integrations.
- **Scoped RBAC**: JWT-based authentication with granular access scopes (e.g., `READ_MEMORY`, `RUN_TASKS`).

### 4. 🚀 Optimization Layer
- **Deduplication**: MD5-based hashing of payloads prevents redundant task execution.
- **Result Caching**: Redis-backed cache for high-frequency sub-tasks.
- **Billing Guard**: Real-time usage tracking and rate-limiting per subscription tier.
